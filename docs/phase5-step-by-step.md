# Phase 5: 実機移行 ステップバイステップガイド

ハードウェア初心者が、シミュレーション完成済みの grass-chopper を実機に載せるまでのロードマップ。
**各ステップは前のステップの成果物に積み上げる構成**で、途中で止めても無駄にならない。

---

## 全体像

```
Step 0  モーターを回す          ← 電気の基本を体感
Step 1  Pico でモーター制御     ← プログラムでモーターを動かす
Step 2  2輪で走らせる           ← 差動駆動の理解
Step 3  Pi 5 と接続             ← ROS 2 と実ハードウェアの橋渡し
Step 4  LiDAR で SLAM           ← シミュレーションと同じコードが動く体験
Step 5  Nav2 で自律走行         ← ゴール指定で自動移動
Step 6  草刈り機構              ← ナイロンコード + BLDC モーター
Step 7  屋外対応                ← 防水・バッテリー・GPS
Step 8  フルミッション          ← シミュレーションの全機能を実機で再現
```

---

## Step 0: モーターを回す

**目的**: 電気でモーターが回る、という体験をする。コードは書かない。

### やること

1. DC モーターの 2 本の線に電池をつなぐ → 回る
2. 線を逆にする → 逆回転する
3. モータードライバ (L298N) 経由で回す → 速度が変えられることを確認

### 学べること

- 電圧・電流の基本 (電池 4 本 = 6V)
- モータードライバの役割 (マイコンの弱い信号でモーターを回す)

## Step 3: Pi 5 と接続 — ROS 2 の世界へ

**目的**: Pi 5 で ROS 2 を動かし、Pico と通信して ros2_control でモーターを制御する。

### 前提: Step 2 完了

### やること

1. Pi 5 に Ubuntu 24.04 + ROS 2 Jazzy をインストール
2. Pi 5 ↔ Pico を USB/UART で接続
3. ros2_control の hardware_interface を実装 (UART で Pico と通信)
4. diff_drive_controller を設定
5. `ros2 topic pub /cmd_vel` でロボットが動く！

### 追加で買うもの

| 品目 | 数量 | 価格 | 購入先 |
|------|------|------|--------|
| Raspberry Pi 5 (8GB) | 1 | ¥13,000 | Amazon / スイッチサイエンス |
| microSD カード (64GB) or NVMe SSD | 1 | ¥1,500 | Amazon |
| Pi 5 用電源 (27W USB-C) | 1 | ¥2,000 | Amazon |
| **追加合計** | | **¥16,500** | |
| **累計** | | **¥31,000** | |

### ここで起きやすいハマりどころ

| 問題 | 対策 |
|------|------|
| `use_sim_time` が True のまま | 実機では全ノード `false` に統一 |
| URDF の `<ros2_control>` ブロック不一致 | joint 名を URDF のリンク名と厳密に一致させる |
| diff_drive_controller が TwistStamped を要求 | Jazzy では `use_stamped: true` が必要 |
| Pi 5 の電源不足でリブート | 27W (5V/5A) 電源を使う、モーター電源は別系統 |

### 完了条件

- [ ] `ros2 topic pub /cmd_vel geometry_msgs/msg/Twist ...` でロボットが前進する
- [ ] `/odom` トピックにエンコーダベースのオドメトリが流れる
- [ ] `robot_state_publisher` が TF ツリーを発行する

### 所要期間: 1〜2 週間

## Step 5: Nav2 で自律走行

**目的**: ゴール地点を指定すると自動で移動する。grass-chopper の Nav2 設定がそのまま使える。

### 前提: Step 4 完了

### やること

1. nav2_launch.py を実機用に調整 (use_sim_time: false)
2. NavigateToPose でゴール指定 → 自動移動
3. Collision Monitor で障害物停止を確認
4. カバレッジ走行テスト (小さな領域: 2m × 2m)

### 追加で買うもの

なし (Step 4 までのハードウェアで可能)

### 調整が必要な設定

| パラメータ | シミュレーション値 | 実機推奨値 | 理由 |
|-----------|-----------------|-----------|------|
| use_sim_time | true | **false** | 実時刻を使う |
| max_vel_x | 0.25 m/s | **0.15 m/s** | まず安全な速度で |
| xy_goal_tolerance | 0.25m | **0.3m** | 実機はオドメトリ誤差が大きい |
| inflation_radius | 0.4m | **0.5m** | マージンを広めに |

### 完了条件

- [ ] NavigateToPose で 2m 先のゴールに自動到達
- [ ] 障害物を置くと Collision Monitor で停止する
- [ ] coverage_commander_node で 2m × 2m をジグザグ走行

### 所要期間: 1〜2 週間
